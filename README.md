## Description

A fully dynamic, single-page restaurant website with tab-based navigation. All content is generated with JavaScript — no hardcoded HTML inside the body. Built around the concept of a cozy café that combines specialty coffee, artisan desserts, and board games.

### Features

- Tab navigation (Home, Menu, About, Contact) with dynamic content rendering
- Full-screen hero section with background image and overlay
- Menu organized by categories: Specialty Coffees, Artisan Desserts, and Experiences
- Elegant dark aesthetic with gold accents
- Google Fonts: Sacramento, Cormorant Garamond, Josefin Sans
- Sticky header with glassmorphism effect
- Responsive card grid for menu items
- Deployed to GitHub Pages via `gh-pages` branch

## Project Structure

```
restaurant-page/
├── src/
│   ├── index.js        — entry point, tab navigation logic
│   ├── template.html   — base HTML template
│   ├── styles.css      — all styling

│   ├── menu.js         — Menu tab module
│   ├── about.js        — About tab module
│   └── contact.js      — Contact tab module
├── dist/               — compiled output (generated by Webpack)

```

## How to run locally

1. Clone the repository:
```
git clone https://github.com/jesuspriv/pagina_restuarante.git
```

2. Install dependencies:
```
npm install
```

3. Start the dev server:
```
npx webpack serve
```

4. Open `http://localhost:8080` in your browser.

## What I learned

This project introduced **Webpack** and **ES6 modules** for the first time, requiring a fundamentally different way of organizing code.

Key concepts practiced:

- **Webpack configuration** — entry point, output, HtmlWebpackPlugin, style-loader, css-loader, asset/resource loader
- **ES6 modules** — `export` and `import` to split code into separate files by responsibility
- **DOM manipulation** — all content is created programmatically with `createElement`, `classList`, `innerHTML` and `appendChild`
- **Tab-based SPA pattern** — clearing `#content` and rendering the correct module on each nav click
- **GitHub Pages deployment** — using `git subtree push` to deploy the `dist/` folder to a `gh-pages` branch
